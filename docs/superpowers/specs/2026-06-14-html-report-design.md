# HTML Export Report — Design Spec
Date: 2026-06-14

## Overview

Replace the current CSV export with a polished self-contained HTML report. The file is fully inline (CSS + data embedded), opens in any browser, and is suitable for emailing. Audience is both internal team and external executives.

## Trigger

The existing "Export report" button generates and downloads the HTML file named `dashboard-report-DD-MM-YY-HH-MM.html`.

## Report Structure

### Header
- Title: "Project Manager Dashboard — Status Report"
- Generated date and time (formatted as `DD/MM/YY HH:MM`)
- Two summary stat boxes:
  - **Total Projects** — count of all projects
  - **At Risk** — count of projects where health is Yellow or Red (orange color if > 0)

### Section 2: At Risk Projects
- Heading: "Projects At Risk"
- Sorted: Red projects first, then Yellow
- Each row: Customer name | Project name | Health pill (colored) | Risk Reason text
- If no at-risk projects: show "No projects currently at risk."

### Section 3: Newly Added Projects
- Heading: "Newly Added Projects"
- Definition: projects that exist in the current `projects` array but whose `name` does not appear in `backups[0].projects` (most recent backup)
- If `backups` array is empty OR no new projects found: section is hidden entirely
- Each row: Customer name | Project name | PM | Start date | End date

### Section 4: Filter Bar
- Three inline dropdowns rendered in the HTML with client-side JS:
  - PM (populated from all unique manager values in the data)
  - Health: All / Green / Yellow / Red
  - Progress: All / 0–39% / 40–69% / 70–100%
- Filters apply in real time to the All Projects table (Section 5)
- Filter dropdowns persist their state when the user interacts

### Section 5: All Projects Table
- Collapsed by default — a toggle button "▶ Show all projects (N)" expands/collapses
- When expanded, projects are grouped by PM (same as dashboard)
- Columns: Customer | Project | NRR(h) | Start | End | Health | Progress | Project Status | Manager Notes
- No Jira URL column, no Actions column
- Health pills and progress bars styled with inline CSS matching the dashboard
- Filters from Section 4 apply — hidden rows are display:none, not removed

## Visual Style
- Dark navy theme matching the dashboard (`#07111f` background, `#0f172a` panels)
- Inter font (loaded from Google Fonts)
- Health pills: Green = green tint, Yellow = amber tint, Red = red tint
- Progress bars rendered as inline HTML divs
- Print-friendly: `@media print` hides filter bar and collapse toggle, expands the projects table

## Technical Notes
- All data is serialised as a JS `const` at the top of the `<script>` block inside the HTML file
- Filtering is pure client-side JS — no external dependencies
- The file is generated by a new `generateHTMLReport()` function in `app.js` that replaces the current CSV export handler

## Affected Files
- `app.js` — replace `exportBtn` click handler with `generateHTMLReport()`
